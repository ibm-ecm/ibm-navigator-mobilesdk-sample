The following enumerations are available globally.

  • IBMECMAccessControlType

    Enumeration of the access control type, which specifies the security access (allow, deny, or delegate) that a user has for a given object.

    • ALLOW: The user or group has the given access to a specified object.
    • DENY: The user or group does not have the given access to a specified object.
    • DELEGATE: Access is delegated to a proxy object. The effective permissions of the object are that of the proxy object.

    • USER: The grantee is a user.
    • GROUP: The grantee is a group.

  • IBMECMAccessControlSource

    Enumeration of the source of the access control entry.

    • CLASS: The access control entry is assigned to an instance of the class as a result of default settings on the class.
    • DIRECT: The access control entry is assigned directly on an instance of the class.
    • POLICY: The access control entry is assigned according to a policy that controls access to an object when its state changes.
    • INHERITED: The access control entry is inherited from the parent.
    • SECURITY_PROXY: The access control entry originates from a security proxy.

  • IBMECMAccessInheritableDepth

    Enumeration of the maximum depth to which an access can be granted.

    • NO_INHERITANCE: No inheritance (this object only).
    • THIS_AND_IMMEDIATE: This object and immediate children only.
    • THIS_AND_ALL_CHILDREN: This object and all children (infinite levels deep).
    • ALL_CHILDREN_BUT_NOT_THIS: All children (infinite levels deep) but not this object.
    • IMMED_CHILDREN_BUT_NOT_THIS: Immediate children only; not this object.

  • IBMECMContentItemVersionStatus

    Enumeration of a content item version status.

    • Released: The version is released. A released version is typically made available to all users. Only one version in a version series can be in the released state at a time.
    • InProcess: The version is in process. An in-process version is typically a draft that is made available to a restricted set of authors and reviewers.
    • Reservation: The version is reserved or checked out. A reserved version can’t be changed by other users.
    • Superseded: The version is superseded. A superseded version is no longer the most recent version. A more recent version is available.
    See more
